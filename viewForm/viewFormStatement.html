Sharing Codes Between View and Data Entry Forms

In Mfg shop environments, viewing data forms are used to support daily production operations. With web applications, viewing data forms can be simply just data entry forms with a view only CSS class. It is simple to add view only CSS class to existing data entry forms.
The benefits of doing this are
  • Reduce redundancy codes
  • Time, Cost and Quality
  • Maintain codes in one place -- it is same concept like adding code into a lookup table in database.
Below is an example of using view only CSS class with a data entry form (load a data entry form, remove submit button and add CSS class): 
     /*View Only CSS content */
           .viewOnly {
               pointer-events: none;
               cursor: not-allowed;
           }
     <!--HTML codes before loading.-->   
     <section id='content'>
          <div id='btnDiv'>
               ... submit buttons group goes here
          </div>
          <div>
               ... data entry form goes here
          </div>
          </section>
     <!--load data entry from to a web page -->
     $("#content").load("your HTML document.html");
     <!--Remove buttons elements-->
     $("#btnDiv").remove();    
     <!--Add viewOnly class to parent of the form-->
          $("#content").wrapInner("<div class='viewOnly'></div>")
          <!--The result in HTML code after it has been loaded-->
          <section id='content'>
          <div class='viewOnly'>
               ... data entry form goes here
               ...
          </div>
          </section>
     <!--To do these steps in one line in Jquery-->
     $('#content').load("yourDataEntryForm.html", function (data) {
          $("#btnDiv").remove();
          $("#content").wrapInner("<div class='viewOnly'></div>");
     });
