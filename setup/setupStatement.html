Setup

It is easy and simple to implement this into a website. In the header section, it requires 4 files, brandColor.css, customColors.css, brandColor.js and initBrandColor.js files.
Create a css file to incorporate variable colors to the Bootstap framework. It is time consuming. For example of CSS codes needed to change from (in Bootstrap.CSS)
.form-control {
...
border: 1px solid #ced4da;
}
To (in Site.css or your css file)
.form-control {
border-color: rgba(var(--todayColor), var(--alpha300) );
}
 --todayColor contains rgb color code (red,green,blue) and --alpha300 contains opacity value (in this case it is .3).
Note: Site.css contains duplicated class names used in the Bootstrap frameworks with different colors. the customColors.css contains daily theme class names (not associated with BootStrap framework).
for example
// CSS border color for Sunday and button out-line
.bdSunColor {
border-color: rgb(var(--sunColor));
}
 // CSS backgroud color for Sunday
.bgSunColor {
background-color: rgb(var(--sunColor));
}
.btn-outline {
background-color: white;
color: rgb(var(--todayColor));
}
<button type="button" class="btn btn-outline">Submit</button>
<p class='bgSunColor'>this is a test</p> // e.i the background color of p tag is Sunday Color.
<p class='bdSunColor'>this is a test</p> // e.i the border color of p tag is Sunday Color.
BrandColor.css file is for storing all of color variables or properties generated by the brandcolor.js.
